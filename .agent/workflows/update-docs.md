# 更新文档流程

## 适用场景
- 功能开发完成后同步文档
- 架构调整后更新技术方案
- 发现文档与代码不一致

## 需要更新的文档

### 开发过程中
- `Doc/开发计划.md` - 更新任务状态
- `Doc/技术方案.md` - 记录技术决策

### 重大变更后
- `Doc/PRD.md` - 需求变更
- `.agent/rules/project-overview.md` - 项目结构变化

## 更新原则
1. **及时性**: 代码变更后立即更新
2. **准确性**: 与代码实际情况保持一致
3. **简洁性**: 避免冗余描述

## 任务状态标记
```markdown
- [ ] 待开始
- [/] 进行中
- [x] 已完成
```
